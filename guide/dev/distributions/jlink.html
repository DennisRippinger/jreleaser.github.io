Jlink

Distributions of this type conform to the following file structure:

.
├── LICENSE
├── README
├── bin
│   ├── app
│   ├── app.bat
│   ├── java
│   └── keytool
└── conf
│    └── /* Java runtime */
└── legal
│    └── /* Java runtime */
└── lib
│    └── /* Java runtime */
└── release

Where:

  • the bin folder contains executable launcher scripts.

  • the lib folder contains all required JARs.

Jlink distributions do not require a suitable Java runtime to be installed on the consumer side as they already bundle a Java runtime, as a consequence you must specify a value for their platform. They are usually packaged as a Zip file.

Creating a Distribution

We recommend using the following plugins to create suitable Java binary distributions:

Maven

  • maven-jlink-plugin: creates a modular Java Runtime image.

Gradle

  • badass-runtime-plugin plugin: use it for non-modular applications. Generates distribution files that follow the expected structure. Also, packages the distribution as a Zip.

  • badass-jlink-plugin plugin: use it for modular applications. Generates distribution files that follow the expected structure. Also, packages the distribution as a Zip.
