Packagers

After a release is created at the desired remote Git host, JReleaser can generate and publish specialized packagers into their respective distribution media, such a Git repository you have access to, or a packager store.

The sections defined here can be applied globally (for all distributions) or per distribution. Global configuration is inherited by distribution configuration. Here’s for example how Homebrew and Scoop can be activated for all distributions:

  • YAML

  • JSON

  • Maven

  • Gradle

packagers:
  brew:
    enabled: true
  scoop:
    enabled: true
{
  "packagers": {
     "brew": {
       "enabled": true
     },
     "scoop": {
       "enabled": true
     }
  }
}
<jreleaser>
  <packagers>
    <brew>
      <enabled>true</enabled>
    </brew>
    <scoop>
      <enabled>true</enabled>
    </scoop>
  </packagers>
</jreleaser>
jreleaser {
  packagers {
    brew {
      enabled = true
    }
    scoop {
      enabled = true
    }
  }
}

Given this setup here’s how a distribution may inherit that configuration and disable Scoop while keeping Homebrew active:

  • YAML

  • JSON

  • Maven

  • Gradle

packagers:
  brew:
    enabled: true
  scoop:
    enabled: true

distributions:
  app:
    scoop:
      enabled: false
{
  "packagers": {
     "brew": {
       "enabled": true
     },
     "scoop": {
       "enabled": true
     }
  },
  "distributions": {
    "app": {
      "scoop": {
        "enabled": false
      }
    }
  }
}
<jreleaser>
  <packagers>
    <brew>
      <enabled>true</enabled>
    </brew>
    <scoop>
      <enabled>true</enabled>
    </scoop>
  </packagers>
  <distributions>
    <app>
      <scoop>
        <enabled>false</enabled>
      </scoop>
    </app>
  </distributions>
</jreleaser>
jreleaser {
  packagers {
    brew {
      enabled = true
    }
    scoop {
      enabled = true
    }
  }
  distributions {
    app {
      scoop {
        enabled = false
      }
    }
  }
}
